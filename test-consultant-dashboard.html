Note about Firebase Authentication

The consultant dashboard requires Firebase authentication to be properly configured. Currently, the dashboard expects:

  • A logged-in user with role 'developer'
  • User status must be 'approved'
  • Firebase secrets to be configured in Replit

If authentication isn't working, you'll be redirected to login.html

Dashboard Features

  • Overview Tab: Recent projects, performance metrics, success rate
  • Profile Tab: Consultant information, bio, specialties
  • Projects Tab: Complete project history with filtering
  • Earnings Tab: Monthly, yearly, and lifetime earnings tracking

Data Structure Expected

The dashboard expects the following Firestore collections:

  • users: Consultant profiles with role, status, and personal info
  • projects: Project data with assignedConsultant field linking to user ID
